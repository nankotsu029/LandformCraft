# V2-6-05 Fluid and gravity containment preflight

## Task Index全文

- 目的／前提: apply前にfluid/gravity/neighbor updateがeffect envelope外へ出ないことを判定する。前提は`V2-6-02`と`V2-6-04`（完了）。
- Scope／成果物: content classifier、support/closure rules、boundary seals、unsupported-state denylist、containment evidenceとhard rejectionを実装する。
- 非Scope: apply、settle simulation、事後rollbackによる代替。
- 主変更: `core.v2.placement.safety`、`format.v2.minecraft` metadata、policy Schema、security／placement docs。
- 必須test／D/M/S: contained/uncontained water/lava/sand/gravel/neighbor-sensitive states、classification order不変、bounded scan/cache budget、unknown block state/version/envelope gap拒否。
- Gate: effect上限を証明できないcontentをapply前に100% hard rejectしsnapshot外検出へ依存しなければ完了し、security／placement／roadmapを更新して`V2-6-06`へ進める。停止: 安全な上限が定義できなければ対象capabilityを拒否したまま停止。

## 設計判断

- Journalに新stateは追加しない。`SNAPSHOT_COMPLETE`のまま、sealed `PlacementContainmentEvidenceV2`をapply許可の追加証拠とする（V2-6-06が要求）。
- `format.v2.minecraft.PlacementBlockPhysicsCatalogV2`（`release-2-placement-block-physics-catalog-v1`）がidentifier単位の閉じた分類（SOLID／AIR／FLUID／GRAVITY／NEIGHBOR／UNSUPPORTED）。unknownはfallbackせず拒否。
- Policy `release-2-placement-containment-policy-v1`はenvelopeのphysics policy半径と整合するclosure／support／seal／denylist規則をversion固定。
- Post-apply predicted world = mutation AABB内はplanned overlay、それ以外のeffect envelopeは既存world（snapshot gateway read）。applyは呼ばない。
- FLUID: connected componentをair+fluidでenvelope内flood-fillし、envelope外へ隣接するair escapeがあれば`UNCONTAINED_FLUID`。
- GRAVITY: 下方へ`gravityFallBlocks`以内にSOLID supportが必要。落下経路・着地がenvelope外なら`UNCONTAINED_GRAVITY`。
- NEIGHBOR: neighbor半径内の全cellがenvelope内、かつcatalogがNEIGHBORとして受理する閉じた集合のみ。chain反応し得るstateはUNSUPPORTED denylist。
- Evidence artifactはCONTAINEDのみseal。失敗はexception（evidenceなし）。Schema: `placement-containment-policy-v2.schema.json`＋`placement-containment-evidence-v2.schema.json`。
