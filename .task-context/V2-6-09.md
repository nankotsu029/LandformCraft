# Task capsule V2-6-09 Undo

## Task Index全文（要点）
- 目的: 完了済みRelease 2 operationをconfirmation付きで安全にUndoする。前提 `V2-6-08`（完了済み）。
- Scope: operation-bound Undo plan/confirm、current-world preflight、snapshot restore、settle/full verify、journal retention transition。
- 非Scope: v1 Undo変更、startup Recovery、force overwrite。
- 主変更: `core.v2.placement.undo`、Paper command/service v2 explicit path、journal Schema、tests／user/admin docs。
- 必須test/D/M/S: actor/replay/expiry/world drift/disk/cancel/late completion、canonical restore order、bounded scan/retention budget、tampered/missing snapshot拒否、v1 Undo回帰。
- Gate: world driftを黙って上書きせずfull verify、v1意味不変。docs更新後 `V2-6-10`（着手しない）。
- 停止: force semanticsが必要になった場合。

## 設計決定
- 入力: terminal `APPLIED` journal（全tile `VERIFIED`）＋sealed verify evidence／snapshot／envelope／apply plan。
- Undo confirmationはapply planを書き換えない（plan checksum変更はsnapshot/evidence bindingを壊す）。別契約`PlacementUndoPlanV2`へUNDO confirmationを載せる。
- prepare: UNDO reservation（同placementIdでlease置換）→UNDO confirmation発行→UndoPlan seal→journalはAPPLIEDのまま（confirmation準備メッセージ）。
- execute: actor/expiry/replay検証→confirmation consume→current-world drift preflight（expected applied streamとexact比較、drift時mutationゼロ拒否）→published snapshot strict再検証→逆順restore（`UNDOING`＋canonical RESTORED suffix、VERIFIED prefix）→bounded settle→baseline exact verify→terminal `UNDONE`＋lease解放。snapshotは削除せずretention対象へ遷移。
- journal invariant追加: UNDOINGは{VERIFIED} prefix + RESTORED suffix、UNDONEは全RESTORED。
- Paper: `PlacementUndoApplicationServiceV2`を明示v2 pathとして追加し、commandがv2 journal存在時にdispatch。
