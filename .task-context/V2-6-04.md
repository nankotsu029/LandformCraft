# V2-6-04 Snapshot-all

## Task Index全文

- 目的／前提: 最初のapply前に全effect envelopeをsnapshotしてstrict verifyする。前提は`V2-6-03`（完了）。
- Scope／成果物: canonical envelope order、snapshot artifact/index/checksum、disk budget、all-before-any-apply state invariant、cancel/shutdown cleanupを実装する。
- 非Scope: block apply、settle、rollback execution、Undo UX。
- 主変更: `core.v2.placement.snapshot`、`worldedit` gateway contract、snapshot Schema、tests／operations docs。
- 必須test／D/M/S: multiple envelope/snapshot failure/disk full/cancel/restart、order/thread不変、snapshot/buffer/disk admission、partial/index/path/checksum/TOCTOU拒否、apply gateway未呼出確認。
- Gate: 全snapshot strict read-back後だけstateがAPPLY_READY（=`SNAPSHOT_COMPLETE`）になり失敗時canonical partialなしなら完了し、placement／operations／roadmapを更新して`V2-6-05`へ進める。停止: tileごとのsnapshot→applyへ戻す必要があれば停止。

## 設計判断

- Journal enum（V2-6-01凍結）に`APPLY_READY`は無く`SNAPSHOT_COMPLETE`がapply-ready状態。enum/Schemaは変更せず`SNAPSHOT_COMPLETE`=APPLY_READYとし、`PlacementJournalV2`のstate invariantを強化（SNAPSHOTTING=全tile PENDING、SNAPSHOT_COMPLETE=全tile SNAPSHOTTED+bytes>0）。
- Gateway契約は v1 `core.PlacementWorldGateway` 前例に合わせ `core.v2.placement.snapshot.PlacementWorldGatewayV2` へ置く（WorldEdit/Paper adapterが後続Taskで実装）。readRegion（canonical X→Z→Y stream）とapply系メソッドを分離し、snapshot-allはapplyを絶対に呼ばない。
- Snapshot fileはpalette+VarInt binary（`release-2-placement-snapshot-file-v1`）。2回走査（palette構築→index書出）し、両passのcanonical stream SHA-256一致でworld drift（TOCTOU）検出。palette上限4096、固定buffer。
- Index artifact `PlacementSnapshotPlanV2`（`release-2-placement-snapshot-v1`）: placement/envelope/reservation/confirmation binding、tileごとfile+artifact checksum+semantic (stream) checksum、totals、budget、canonicalChecksum。Schema `placement-snapshot-plan-v2.schema.json`。
- staging `snapshotsRoot/.staging-<placementId>/` → 全file strict read-back → index書出+read-back → atomic move → `snapshotsRoot/<placementId>/`。失敗/cancel/shutdownはstaging全削除、journalはCONFIRMATION_ISSUEDのまま。
- disk admission: reservation.diskLease.reservedBytes上限で実書込bytesをhard stop、事前にestimate admission、safety store leaseのassertOwned。
