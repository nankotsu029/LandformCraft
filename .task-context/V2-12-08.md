# Task capsule V2-12-08 — v2 request authoring

補助資料。正本は `docs/design-v2/task-index.md` と `docs/adr/0035-v1-retirement-governance.md` D11。Git管理外。

## 1. Task Index 対象Task（要約）

- 目的: `V2-12-05`監査のF1解消。v1にしか無いrequest authoringをv2へ。
- Scope: v2 request storeと`v2 request`のauthoring verb（Paper: create／bounds／bounds selection／prompt／list、CLI: create／bounds／prompt／list）。`validate`／`info`不変。Paperは`data/v2/`へsandbox化。strict schema検証→atomic publish。permission細分。
- 非Scope: `GenerationRequestV2`契約変更、AI provider呼び出し、v1 request store変更、default routing切替、image入力経路。
- Gate: v1で可能だったauthoring操作がすべてv2で実行でき、`v2 export`まで接続することをE2Eで示す。

## 2. 実装した形

- `core.v2.command.V2RequestStoreV2`（Bukkit非依存、状態を持たない）
- verb: `create` / `bounds` / `selection`（Paper専用）/ `constraint-map` / `prompt`（Paper=chat取込、CLI=inline）/ `list`
- Paper: `PaperV2WorkflowServiceV2` に authoring API、`LandformCraftCommand` に dispatch と `PromptSession.v2` フラグ
- CLI: `LandformCraftCli.authorRequest(...)`、store root は `<--data-dir>/v2/requests`
- router: 同一 `(verb, operation)` の surface 別形を caller surface → arity で解決

## 3. Scope明確化（Gateへ記録済み）

1. Paperの選択範囲取込は `bounds selection <id>` ではなく `v2 request selection <id>`。routerが`(verb, operation)`で選んでからarityを見るため、同じ`bounds` operationに9 tokenと5 tokenを置くと衝突する。
2. 非Scope「image入力経路」はV2-7の`format.v2.constraint.extract`を指し、request levelのconstraint map **source宣言**は含まないと解釈。`CoastalSurfaceExportPipelineV2.writeConstraintFields`が「宣言ちょうど1件」をfail closedで要求するため、宣言verbが無いとGateの「`v2 export`まで接続」を満たせない。
