# Task capsule V2-12-09 — v2 job, candidate, and export lifecycle

補助資料。正本は `docs/design-v2/task-index.md` と `docs/adr/0035-v1-retirement-governance.md` D11。Git管理外。

## 1. Task Index 対象Task（要約）

- 目的: `V2-12-05`監査のF3解消。v2の`generate`／`export`を非同期job lifecycleへ載せ、`job` verbのv2 backendとcandidate列挙を提供する。
- Scope: v2 job store（job ID／state／cancel token）、非同期実行と既存同期形の維持、`job status`／`cancel`のv2 dispatch、request単位のcandidate列挙、`export`のreservation-bound confirmation token。cancelはcancel token・Future cancel・interrupt・job stateを連動させ中途artifactをpublishしない。
- 非Scope: Release format 2の変更、placement lifecycleの変更、v1 job storeの削除・変更、default routing切替。
- Gate: v1の`generate`→job→`candidate`→`export plan`／`create`相当がv2で完走し、`job status`／`cancel`がv2 jobへdispatchすれば完了。

## 2. 実装した形

- `model.v2.job`: `ExportJobStateV2`／`ExportJobKindV2`／`ExportJobSnapshotV2`
- `schemas/generation-job-v2.schema.json` ＋ `examples/v2/job/export-job-v2.json`（v1 `generation-job.schema.json`は凍結のまま）
- `format.v2.job.ExportJobCodecV2`（strict schema＋staged read-back＋atomic move）
- `core.v2.job`: `ExportJobStoreV2`（durable、上限4096、canonical UUIDのみ）／`ExportJobServiceV2`（submit／status／cancel／candidates）／`ExportJobSubmissionV2`／`ExportPlanStoreV2`（hash保持・constant-time・単回・10分・actor束縛・上限256）
- verb: `v2 export plan`／`v2 export create`（Paper専用）、`v2 job status|cancel|list`／`v2 candidate list|info`（両surface）
- router: 1 verbが直接形とoperation形を併せ持つ場合をoperation名**かつ**arity一致で判定

## 3. 設計判断（Gateへ記録済み）

1. candidate列挙はRelease format 2 manifestがrequest identityを持たない（format変更＝停止条件）ため、job snapshotの`requestId`を正本にした。
2. `export plan`／`create`はPaper専用。planは実行中processに存在し確認したjobも同processで走る（v1も`PaperWorkflowService`のin-memory planでPaper専用だった）。CLIの`job cancel`は別processのworkerに届かずdurable snapshotを`CANCELLED`にするだけ（v1 CLIと同性質）。
3. 上位verb `/lfc job`／`/lfc candidate`のv1／v2 dispatchは実装していない。既定routing切替は`V2-12-05`のScopeで、本Taskはdispatch先のv2 backendを用意した（D10 `job`行の欠落を解消）。
