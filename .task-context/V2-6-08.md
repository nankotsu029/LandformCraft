# Task capsule V2-6-08 Rollback

## Task Index全文（要点）
- 目的: Release 2 placement失敗時にsnapshot済みenvelopeを逆順復元しfull verifyする。前提 `V2-6-04`／`V2-6-06`／`V2-6-07`（完了済み）。
- Scope: reverse-order restore、close/completion tracking、rollback settle/verify、partial failure classification、reservation/journal finalization。
- 非Scope: user Undo（V2-6-09）、startup Recovery（V2-6-10）、snapshot外変更対応。
- 主変更: `core.v2.placement.rollback`、world gateway、journal states、tests／operations docs。
- 必須test/D/M/S: apply・settle・verify各failure pointからのrollback（surface/solid/air/fluid/child/overlay content）、rollback failure／late completion／shutdown、effect envelope全体のreverse restore order不変、bounded slices/disk lifetime、missing/tampered snapshot拒否、v1 rollback回帰。
- Gate: fault injection全点で復元または明示RECOVERY_REQUIREDとなり成功を偽らない。placement／operations／roadmap更新後 `V2-6-09`（着手しない）。
- 停止: snapshot外副作用をrollback対象とする必要が出た場合。

## 設計決定
- 入力: `RECOVERY_REQUIRED` journal（V2-6-06/07の失敗分類）のみ。tile状態は{SNAPSHOTTED, APPLIED}のみ許可。
- preflight（world mutation前に全部）: policy/limits version → journal state/binding → snapshotPlan.requireBindings → safetyStore.assertOwned → `PlacementSnapshotAllCompilerV2.loadPublished` strict再検証（missing/tampered拒否）→ published index checksum == request snapshotPlan checksum → envelope volume admission → baseline構築（全snapshot fileをstrict streaming読取、coord map、overlap一致検査、union envelope被覆完全性、baseline stream checksum）。
- restore: 逆tile-index順、tile内はsnapshotのcanonical X→Z→Y順、`PlacementRestoreSliceV2`（≤4096 blocks、slice数budget）を1枚ずつ受理receipt（schedulerAccepted/mainThread/resourcesClosed）確認後に次へ。tile完了ごとにROLLING_BACK journalへRESTORED suffix checkpoint。
- journal invariant追加: ROLLING_BACKは{SNAPSHOTTED|APPLIED} prefix + RESTORED suffix（canonical逆順checkpoint）、ROLLED_BACKは全RESTORED。
- settle/verify: `PlacementSettleVerifyPolicyV2`を再利用。settle tick→quiescence/timeout、envelope外update拒否。verifyはunion effect envelope全体をreadVerifySliceでexact比較（expected=snapshot baseline）、stream checksum一致。sampled verify禁止。
- 成功時のみ: terminal `ROLLED_BACK` → `FilePlacementSafetyStoreV2.release(placementId)`。published snapshotは削除しない（evidence保持、retention/Undoは別Task）。
- 失敗時: RECOVERY_REQUIRED再seal、部分RESTORED checkpointを保持。late completionは無視（finished flag）。shutdown/cancelもRECOVERY_REQUIRED。
- gateway: `PlacementWorldGatewayV2`へdefault `restoreBlockSlice(PlacementRestoreSliceV2)`追加。
- codec: `PlacementSnapshotFileCodecV2.readStrict`へblock-state sink付きoverload追加（既存挙動不変）。
