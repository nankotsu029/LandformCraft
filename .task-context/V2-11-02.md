## 着手時点の欠陥（監査判定#5の実体）

1. `Landformcraft.java`が`requireRange(..., 1, 10_000, ...)`で`measured-candidate-max-*`を受理していた。catalogの64×64は事実上の推奨値でしかなく、設定で最大10,000まで広げられた。
2. `Release2PlacementApplicationServiceV2`の6引数便宜constructorが`Release2MeasuredDimensionGateV2.unlimited()`を既定にしていた。
3. R2は`disk.maximum-snapshot-bytes`しか受け取らず、`disk.minimum-free-bytes`／`disk.safety-margin-bytes`が無効。reservation／snapshot／Undoのadmissionは固定`MINIMUM_FREE_BYTES`（1 MiB）を使用（4箇所）。
4. `FilePlacementSafetyStoreV2.rebuild`がregion leaseを`plan.target()`箱から再構築していた（三項演算子の両枝が同一のplaceholder）。effect envelopeはmutation箱をcontainment radiiで拡張したものなので、restart後にhalo分が未予約となり第三者がoverlap配置をbindできる。

## 実装

- `PlacementDimensionLimitV2`: inline `10_000`を`CATALOG_BUDGET_MAXIMUM`定数化（新規hard-codeではなく既存値の命名）。
- `Release2MeasuredDimensionGateV2`: `production()`／`production(w,l)`を追加。設定値をcatalog `SMOKE_MEASURED_MAXIMUM`でクランプし、超過は起動時に`IllegalArgumentException`。
- `Release2MeasurementProfileV2`（新規）: `enabled`／`isolatedWorldName`／`maximumWidth`／`maximumLength`。無効時は世界名・上限を持てない。`admits`は flag＋隔離world一致＋`PlacementActorKindV2.CONSOLE`＋宣言上限内をすべて要求する。CONSOLEはRCON/consoleを表し、in-game Playerは到達不能。
- `Release2PlacementDimensionPolicyV2`（新規）: 唯一のadmission point。通常gate→measurement profileの順で判定し、両方外なら安定messageで拒否。
- `Release2DiskBudgetV2`（新規）: 3 disk設定を保持し`reservationFloorBytes() = minimumFree + safetyMargin`。`legacy(...)`は既存caller向けに従来の1 MiB floorを維持（既定を弱めない）。
- `FilePlacementSafetyStoreV2`: `reservationFloorBytes`を注入可能化し、reservation／snapshot／Undo／cleanupの4箇所が定数ではなくこれを使用。
- `FilePlacementSafetyStoreV2.rebuild`: 入力を`RebuildEntryV2(journal, envelope)`へ変更。region leaseはenvelope tileの`effectAabb`列、disk leaseはenvelope disk estimateを下限とする。plan checksumではなくplanの`envelopeReferences`でbindする（journalのplanはreservation／confirmation bindingで再sealされ、envelopeのsource plan checksumとは一致しないため）。consumed confirmation setは保持（落とすとconfirmation replayを許す）。
- `Release2PlacementApplicationServiceV2`: `dimensionGate`→`dimensionPolicy`、production constructorに`Release2DiskBudgetV2`＋policyを追加。`prepare()`のlayout読取直後に`requireAdmitted(w, l, worldName, actor.kind())`。
- `Landformcraft.java`: `release2DiskBudget()`／`release2DimensionPolicy()`を追加。measurement profile有効時はstartup警告。
- `config.yml`: `measurement-profile`ブロックと注記を追加。

## 意図的にやらなかったこと

`inspectRestartState()`から`rebuild()`を呼ぶ配線は試したうえで戻した。rebuildのstate filterはplanned／activeだけを残すが、R2では`APPLIED` placementもUndoまでleaseを保持するため、restart毎にrebuildすると`APPLIED`のleaseが消え第三者overlapを許してしまう（`restartBetweenConfirmApplyAndUndoKeepsSelfLeaseButRejectsThirdPartyOverlap`が検出）。どのjournal stateがleaseを持つかの変更は本TaskのScope外なので、durable ledgerをrestartの正本のまま残し、rebuildは`rebuildReservationsFromEnvelopes(...)`として明示修復用に公開し、effect envelope基準であることをunit回帰で固定した。
