# V2-5-16 Offline 3D schematic read-back

## Task (Task Index 正本)
- 目的: volumeを含むtile block streamをSponge v3へ出しoffline read-backする。前提は`V2-5-14`、V2-2 schematic基盤。
- Scope: air cavity／fluid／independent solidのstable XYZ export、general VarInt、tile boundary palette、WorldEdit 7.3.19とFAWE-compatible offline inspector test。
- 非Scope: Paper apply、FAWE server smoke、Release capability。
- 主変更: `format.v2.tile`、`worldedit` inspector/adapters、3D fixtures／artifact docs。
- 必須test/D/M/S: cave/overhang/sky/fluid/air, palette boundaries, whole/tile XYZ stream, thread/order不変, stream/palette/buffer budget, truncated/invalid VarInt/checksum改変拒否。
- Gate: offline read-back block streamがresolver checksumと全XYZ一致しv1/V2-2 export回帰不変なら完了→artifact/volumetric/roadmap更新→`V2-5-17`。server runtime必要ならV2-6へ延期。

## 既存基盤（凍結、変更しない）
- `OfflineTileSchematicWriterV2` / `SpongeV3TileInspectorV2` / `WorldEditOfflineTileReaderV2` は既に全Y範囲(minY..maxY)をX-fast/Z/Y順で走査する汎用3D exporter。VarInt汎用、Offset[0,0,0]、palette 1..16384。
- `CanonicalBlockStreamV2` semantic checksum。allowlistは`EnvironmentBlockStateCatalogV2`（air/water/solid含む、127/128境界超集合）。
- `VolumeTerrainQueryV2`(V2-5-05)はvolume-composed `TerrainQuery`（QUERY_KERNEL_VOLUME_V1）。composed sample: blockClass/semanticMaterial/fluidBody。

## 追加する新規コード
- `format.v2.tile.VolumeTileBlockResolverV2`: volume-composed `TerrainQuery`をcanonical block stateへ橋渡しするv2 export adapter。air→air, fluid(WATER)→water, solid→semantic material。QUERY_KERNEL_VOLUME_V1を要求、EnvironmentBlockStateCatalogV2 allowlist経由。v1 adapterと分離。

## FAWE注意
- FAWEはtest classpathに無い（worldedit-core 7.3.19のみ）。「FAWE server smoke」は非Scope。
- 「FAWE-compatible offline inspector」= 出力Sponge v3が一般仕様features（Version3/DataVersion4671/Offset[0,0,0]/general VarInt/proprietary tag無し）だけを使うことをoffline検証。runtime FAWE smokeはV2-6/BLOCKED_EXTERNALのまま。
