# LandformCraft 0.9.0-beta.1 Release Candidate

## Compatibility

- Java: 21
- Minecraft／Paper: 1.21.11
- WorldEdit compile／target: 7.3.19
- WorldEdit確認済み: 7.3.19（64×64 structure Releaseのapply／full verify／Undo）
- FAWE候補: 2.15.2。独立server初期化まで確認したがsandbox port bind制限でcurrent apply smoke未完了
- Schema version: 1
- Generator version: `3.0.0-phase6`
- Sponge Schematic: v3、DataVersion 4671

## Highlights

CLIとPaperからrequest、manual／OpenAI／Anthropic design、6-role画像、deterministic generation、8 preview、8 built-in structure、Release directory／ZIP、verify、snapshot付きplacement、rollback、Undo、Recoveryを操作できます。

beta hardeningではoverlap reservation、actor-bound token、disk reservation、retention cleanup、stable error／correlation、Provider config、制限付きcustom asset catalogを追加しました。独自Web UIは削除しましたが、ブラウザ版AIによるJSON作成は維持します。

## New command groups

Paper: `version`、`doctor`、`request`、`design`、`generate`、`job`、`candidate`、`export`、`apply recover`、`asset`、`cleanup`。

CLI: `version`、`doctor`、`request`、`job status|cancel`、`candidate`、`asset`、`recovery`と、`--data-dir`／`--json`／`--quiet`／`--verbose`。

詳細は [commands.md](../commands.md) を参照してください。

## Data compatibility and upgrade

1. Paperを停止する。
2. worldと `plugins/LandformCraft` data directoryを同時backupする。
3. activeなplan／apply／Undoがないことを確認する。Phase 3のterminal journalは読取り時に`SYSTEM:LEGACY`とunknown byte usageとして安全に移行されるが、activeな旧confirmationはactor bindingへ変換できない。
4. WorldEditまたはFAWEを一方だけ対応版へ更新する。
5. JARを置換し、default configとの差分を人手でmergeする。API key値は書かない。
6. test worldで `/lfc version`、`doctor`、verify、apply／Undo、Recovery dry-runを実行する。
7. 旧journalを新Schemaへ無理に手編集しない。

このbetaが書いたjournal／safety state／cleanup planから古い版へのdowngradeは保証しません。

## Known limitations

- 独自Web UI、entity、block entity、biome書換え、洞窟、巨大都市は非対応
- custom asset catalogは安全な検証・保管に対応するがTerrainIntent v1はasset IDを直接選択しない
- live Provider model availabilityとProvider側exactly-onceは保証せず、成功responseのdurable local cacheも未実装
- error codeとcorrelation IDは安定しているが、JSONのcontext IDは共通`resourceId`でありresource種別ごとの名前付きfieldは未実装
- CLI `job cancel`は別processの実行中ownerへ通知せず、foregroundのCtrl+CとPaper in-process cancelだけが実taskへ伝播する
- domain artifactはconfirmation tokenのhashだけを保存するが、CONSOLE command候補は標準Paper logへ平文記録される
- 1000×1000実world applyの性能はserver hardware／WorldEdit implementation別に保証しない
- release candidate公開前のcurrent WorldEdit／FAWE smokeとperformance記録は [beta checklist](../beta-release-checklist.md) で未完了ならblocker
- 500×500／1000×1000 generate・export・verifyは再計測済みだが、同サイズのPaper plan／snapshot estimate／apply／Undoは未計測

## Release decision

WorldEdit E2Eは成功しましたが、FAWE current apply smoke、500／1000 Paper plan、最新sourceの最終clean build、Provider durable response cache、custom asset生成統合、typed error context、CLI cross-process cancel、CONSOLE tokenのlog非永続化が未完了のため、この文書時点では **Beta blockerが残るため未リリース** です。

production導入前に必ずbackupを取り、同じ構成のtest serverでReleaseを検証してください。
